1. Getting Started

System Requirements

  • Recommended specifications:
    • Windows 7 or later with x64 (64-bit) CPU architecture
    • 1GHz or faster CPU
    • At least 4GB RAM
    • Microsoft .NET 4 (which is available for Windows XP SP3/Windows Server 2003 SP2 and later versions)
  • Minimum specifications:
    • Windows XP SP3 or later with x86 (32-bit) or x64 (64-bit) CPU architecture
    • 1GHz or faster x86/x64 CPU
    • 1GB RAM for x86-architecture systems OR at least 2GB RAM for x64-architecture systems
    • Microsoft .NET 4

Overview

Yahtzee! is a luck/strategy game in which players score points by rolling a set of five six-sided dice.

Objective

The objective is to score as many points as possible from the dice combinations rolled. The player with the highest score at the end of the game is the winner.

2. Gameplay

Introduction

Yahtzee! is a game that employs both chance and strategy. By rolling a set of five 6-sided dice, it is up to you how you wish to award your own points based on the combinations you roll.

Adding/Removing Players

After starting the application and the splash screen has disappeared, you will be greeted with a dialog box with a welcome message and an instruction to add all players who will play the round. To do so, you must click the large plus [+] button on the lower-right side of the dialog box. Each player must be entered with a unique name, each of which may be up to 15 characters in length, and, due to performance limitations (see below for details) up to four players may play in each round. Once you are satisfied with your roster, you may begin playing by clicking the Play! button. Once you do so, the program will generate a score sheet containing score slots for every player entered in the roster. Generating the score sheet may take a moment, depending on the size of the roster and the overall performance of your system.

The following information outlines action events supported in the roster section of the dialog box:

  • MOUSE EVENTS:
    • CLICKING-AND-DRAGGING UNPOPULATED AREA: Allows "lasso" selection for multiple items listed in the roster box.
    • DOUBLE-CLICKING ITEMS: Allows renaming of players added to the roster.
    • RIGHT-CLICKING ITEMS: Displays a context menu containing an option to Rename the selected entry (if only one is selected) and to Delete all selected entries in the roster.
  • KEYBOARD EVENTS:
    • DELETE KEY: Deletes all selected player entries in the roster.

Turn-Based Gameplay

The score sheet will be displayed as a dual-axis table, with the first 15 letters of each player's name displayed across the first row (which we'll call the "queue") and the category labels displayed along the first column.

A player's turn will be indicated by a light yellow column beneath the currently active player's name on the score sheet. Once the active player has used his/her turn, it will rotate to the next player in the queue. After the last player in the queue has used his/her turn, it will return to the first player in the queue. This cycle will repeat until every player has filled every category in his/her column, at which point the game will end and the application will declare the final winner(s).

Rolling the Dice

For each turn, the active player will be given three opportunities to roll the dice, of which five are provided. The number of rolls remaining will be indicated in the caption of the Roll button in the dice roller toolbar at the top.

If you see a pattern respective to any of the categories you are aiming to fill, you have the beneficial option to hold dice to prevent them from being rolled. You can hold or unhold a die by clicking on it. A held die is indicated by a shaded background color with white dots, while an unheld die is indicated by a white background with black dots.

Category Selection

Before concluding one's turn, a player must decide which category to fill based on the dice value combination he/she has rolled, even if the dice combination does not match any available category. More information on how to effectively choose a category can be found in the Scoring section of the documentation.

You cannot select a category before using your first roll. Once you are ready to select a category or have exhausted all your rolls, you can select a category by clicking one of the suggested scores that will temporarily appear on the score sheet, highlighted in red. Once you have done so, the suggestions will disappear and the game will proceed to the next turn.

If you are playing in a multiplayer setting, your turn will rotate to the next player in the queue once you have selected a category.

Scoring

Your score tally will depend on your dice value combination and what category you select based on that combination. If your selected category does not in any way relate to the combination you have rolled, a score of zero (0) points will be awarded to that category. Each category can only be used once per player per round.

The following information outlines how many points you can earn from the categories you choose:

  • UPPER SECTION
    • ACES: Adds one point for every 1 rolled.
    • TWOS: Adds two points for every 2 rolled.
    • THREES: Adds three points for every 3 rolled.
    • FOURS: Adds four points for every 4 rolled.
    • FIVES: Adds five points for every 5 rolled.
    • SIXES: Adds six points for every 6 rolled.
  • LOWER SECTION
    • 3 OF A KIND: Adds the total of all dice, if at least 3 dice show the same value.
    • 4 OF A KIND: Adds the total of all dice, if at least 4 dice show the same value.
    • FULL HOUSE: Adds 25 points for a 2-of-a-kind and a 3-of-a-kind.
    • SMALL STRAIGHT: Adds 30 points for a sequence of 4.
    • LARGE STRAIGHT: Adds 40 points for a sequence of 5.
    • YAHTZEE: Adds 50 points for a 5-of-a-kind.*
    • CHANCE: Wildcard category. Adds the total of all dice, regardless of values rolled.

UPPER SECTION TOTAL: If you score 63 or more points in your Upper Section Subtotal, an additional 35-point bonus will be awarded to your Upper Score Total.

LOWER SECTION TOTAL: * Per traditional game rules, for every subsequent Yahtzee you roll, an additional 100-point Yahtzee Bonus will be awarded to your Lower Section Total. However, Yahtzee Bonus is not currently available in this version of the software.

3. Error Logging

Albeit rare, there may be some occasions in which the application may encounter a fatal error and crash thereafter. For instance, because the application attempts to read and write data to the disk on which it is running, it may, for example, crash in the event that the disk is read-only (such as on an SD card with the switch set to read-only mode), out of space, suddenly removed from the host system (if the storage device is external), or saving to a location where write access is restricted due to lack of administrator/superuser privileges.

In the event of an unhandled program error, the application will display a dialog explaining that the application has crashed and will show technical details describing the cause of the error (this may be useful information for developers). You will be given 20 seconds by default to either Restart the application or Terminate it, after which the application will automatically attempt to restart. You may disable the countdown timer by unchecking the Restart automatically checkbox if you wish to keep it open.

Should you ever be presented with the error dialog, it will generate an error log and save it within the local directory where the executable is located, under \Sidera\Yahtzee\logs. The file will be named with the format errorlogYYYYMMDDHHMMSS.txt, with YYYYMMDDHHMMSS being the error log's timestamp.
